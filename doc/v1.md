# v1.0 — Folding, Clipboard, DAG Links

Post-MVP milestone. Branch: `v1.0-dag`.

## Goal

Extend the outliner from a pure tree to a DAG (directed acyclic graph)
where a single node can appear under multiple parents via link-paste.
This requires three prerequisite capabilities delivered in sequence.

## Data Model Changes

### SiteNode (render tree)

The MVP renders directly from `Graph` — a full recursive walk on every
render. v1.0 introduces `SiteNode`, a persistent render tree that:

- Assigns each **appearance** of a node a stable `instanceId: int`
- Tracks per-appearance `expanded: bool` (fold state)
- Allows the same `NodeId` to appear in multiple locations with
  independent fold states
- Is reconciled (not rebuilt) after graph mutations, preserving
  existing instance IDs

```
type SiteNode =
    { instanceId: int
      nodeId: NodeId
      expanded: bool
      children: SiteNode list }
```

`Model` gains `siteRoot: SiteNode` and `nextInstanceId: int`.

### ClipboardContent

Internal clipboard holds a self-contained snapshot of copied/cut nodes,
independent of `graph.nodes`:

```
type ClipboardContent =
    { topLevelIds: NodeId list
      nodes: Map<NodeId, Node> }
```

`Model` gains `clipboard: ClipboardContent option`.

exactly one parent (no DAG yet), so instance IDs are 1:1 with NodeIds.

- [X] `buildSiteTree : Graph -> SiteNode * int` — initial full build from graph
- [X] `rebuildSiteTree : Graph -> SiteNode -> int -> SiteNode * int` — reconcile

### Phase 2: Cut / Copy / Deep-Copy Paste

- [ ] Unit tests for `serializeSubtree`, `collectSubtree`, `buildPasteOpsFromClipboard`

### Phase 3: Link-Paste + DAG Snapshot

Paste can now reuse existing NodeIds (link-paste) when a program option is
enabled. Snapshot format extended to represent DAGs.

- [ ] Settings checkbox: **"Paste Gambol content as links"** (default off)
- [ ] `onPaste` detects Gambol snapshot format in clipboard; if option is on
      → link-paste (reuses existing NodeIds, `Op.Replace` only);
      otherwise → existing deep-copy path
- [ ] `rebuildSiteTree` handles DAG: linked nodes get new `SiteNode` instances
      (new `instanceId`, default collapsed)
- [ ] Cycle guard: `rebuildSiteTree` tracks ancestor set per path; stops
      recursion if a nodeId appears in its own ancestor chain
- [ ] Extend `Snapshot.write`:
  - Track visited nodes during preorder walk
  - First visit: emit `#shortId text` (shortId = sequential `n1`, `n2`, ...)
  - Subsequent visits: emit `-> #shortId`
- [ ] Extend `Snapshot.read`:
  - `#id text` → create node, register in ID map
  - `-> #id` → look up existing node, add as child
  - Plain lines (no `#` or `->`) → fresh ID (backward compatible)
- [ ] Snapshot round-trip tests with shared nodes
- [ ] Backward compatibility test: old-format snapshots load unchanged
- [ ] `SetText` on a linked node updates all appearances (already works —
      graph stores one `Node` entry per `NodeId`)

## Key Design Decisions

| Decision | Rationale |
|---|---|
| `SiteNode` with `instanceId` as fold key | Per-occurrence fold state; same node in two locations can be independently expanded/collapsed |
| `ClipboardContent` is a standalone snapshot | Independent of `graph.nodes`; survives graph mutations and snapshot reload |
| `copy`/`cut` event handlers instead of key table | `e.clipboardData.setData` is synchronous and permission-free; works on non-HTTPS and Firefox; no key table entries needed for `Ctrl+C`/`Ctrl+X` |
| `serializeSubtree` called in View handler, not `update` | `e.clipboardData` is only valid synchronously during the event; must write before handler returns |
| Single `Ctrl+V` for paste; settings option controls link vs deep-copy | Avoids `navigator.clipboard.readText()` (async, HTTPS-only, permission-gated); no custom keybinding needed |
| Deep-copy = fresh NodeIds, link-paste = reuse NodeIds (Phase 3) | Link-paste is the more common intent in a DAG outliner; deep-copy is the default |
| Snapshot format extended with `#id` / `-> #id` | Human-readable, backward compatible with plain tab-indented files |
| Cycle guard in render, not in data model | The graph is a DAG by construction (paste only links existing subtrees); guard is defense-in-depth |
